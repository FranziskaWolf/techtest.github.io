GTM and data layer

Setting up the GTM and implementing the tag on the site went well. I used the Tag Assistant Legacy for Chrome to check that it works. First step has been done. So now we can come to the tricky part. Using GTM I am able to create different variables, tags and trigger to add values coming from the webiste. So the following variables need to be set up: for example eventAction, eventCategory, eventLabel, eventValue, transactionTotal. After that I needed to set up the tag and trigger. Final step is creating an HTML-tag for the Data Layer Push.

Challenges

The problem now is that I can't really submit any "real" information. So I tried to show you what is supposed to happen. For this I chose a bookshop as an example. Every part of the table is a key: bookAuthor, bookTitle and bookPrice. Every key has different values like "war and peace" or "pride and prejudice". I sat up those variables like mentioned above in my Google Tag Manager. Afterwards I used a script including "dataLayer.push()" to transfer the given information to the data layer.

window.dataLayer = window.dataLayer || [];
dataLayer.push({
'transactionProducts': [{
'bookAuthor': 'J.R.R. Tolkien',
'bookTitle': 'Lord of the rings',
'bookPrice': 29.99,
'quantity': 1
},{
'bookAuthor': 'Jane Austen',
'bookTitle': 'Pride and prejudice',
'bookPrice': 19.99,
'quantity': 2
}]
});

API Implementation

When it comes to setting up an API it's important to think of different requirements like: what kind of information should be shared and who is supposed to get it? How can I make sure the API is performing technically in the best way? How can I control the traffic running through and how much traffic will actually be expected? Is this a customised solution or more standardized and is there maybe a documentation for it, provided by the client?
The endpoint is the URL you provided and my data layer is supposed to send its values there. I think its a a GET script like this: "GET /my-api/my-resource/{name}".
Google Tag Manager provides API solutions ("apis.google.com/js/api"). And although everything is pretty well documented on their site, I haven't been able to set it up right.
